📋 Observations & Notes

  • 28 read-only tools tested. 23 passed cleanly, 3 had caveats (empty results or skipped for valid reasons), 2 had issues (file_download_url error, search_unified partial failure on personal account).
  • calendar_check_availability requires timezone-aware datetimes. Naive ISO strings (without offset) are rejected with a validation error. Always pass +09:30 or Z suffix.
  • file_download_url fails for personal OneDrive accounts when the file ID comes from a search result. The Graph API doesn't always populate the download URL for personal accounts via the search endpoint. Use file_get (write tool) to actually download.
  • search_unified on personal accounts only reliably returns driveItem results. Email and event search appear to return empty arrays regardless of query. This matches the server's documented behaviour: personal accounts fall back to sequential type-specific searches, which may not be wired into the unified endpoint.
  • contact_list ordering is alphabetical by file-as name, not by most recently added. No total count is returned in the response — you need to paginate to discover the full count.
  • emailrules_list returned 78 rules — a comprehensive automated email filing system across personal, bills, technical, services, shopping, and misc categories. Rules make heavy use of sender domain matching and subject keyword patterns.
  • Cache state was "miss" for most first-call results and "fresh" for tools called in sequence after a prior warm call (e.g. contacts, calendars). Cache TTLs vary by resource type (email: 2 min; contacts: 20 min; calendar: 5 min).
  • email_get_attachment was the only tool that couldn't be tested at all — no suitable attachments were present in the inbox at the time of testing. The tool's parameters and error handling remain unverified.
  • folder_get_tree on /JobSearch revealed a git repo being synced to OneDrive (visible .git directory with full object tree). Probably unintentional; worth reviewing OneDrive selective sync settings.
