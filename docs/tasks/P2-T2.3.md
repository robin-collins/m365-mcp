# Phase 2 - T2.3 Completion Report

Task: Centralize outbound attachment validation.

Source: `docs/audit_remediation_task_list.md`, T2.3.

## Summary

Centralized outbound mail attachment preparation so draft creation and sending
both validate local paths before reading files. This closes the direct
`Path(...).read_bytes()` bypass described in audit finding N11.

## Changes Made

- Added `_prepare_outbound_attachments()` in `src/m365_mcp/tools/email.py`.
- Routed `email_create_draft` and `email_send` through the shared helper.
- Enforced `ensure_safe_path`, existence checks, allowed roots, per-attachment
  size checks, and attachment-count limits before reading bytes.
- Added tests for:
  - accepted small attachment payloads,
  - path traversal rejection,
  - disallowed-root rejection,
  - oversize attachment rejection,
  - attachment-count rejection.
