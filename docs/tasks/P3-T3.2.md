# Phase 3 - T3.2 Completion Report

Task: Replace impossible targeted invalidation patterns.

Source: `docs/audit_remediation_task_list.md`, T3.2.

## Summary

Replaced cache invalidation patterns that searched for raw parameter names inside
hashed cache keys. Until cache metadata columns exist, these mutations now
broaden invalidation to the affected resource type for the target account.

## Changes Made

- Broadened sent-mail invalidation from impossible sent-folder text matching to
  `email_list:*` scoped by account.
- Broadened email message invalidation from `email_id=...` matching to
  `email_get:*` scoped by account.
- Broadened OneDrive folder-specific list invalidation from `folder_id=...`
  matching to `file_list:*` scoped by account.
- Added regression tests for `email_update` and `file_rename` cache
  invalidation, alongside the existing delete coverage from T3.1.
