# Phase 5 - T5.1 Completion Report

Task: Add a hard SQLCipher availability guard.

Source: `docs/audit_remediation_task_list.md`, T5.1.

## Summary

Added an explicit SQLCipher availability flag and made encrypted cache
initialization fail clearly when `sqlcipher3` is unavailable. This prevents the
stdlib SQLite fallback from silently running while logging that encryption is
enabled.

## Changes Made

- Added `USING_SQLCIPHER` in `src/m365_mcp/cache.py`.
- Raised `ImportError` when `encryption_enabled=True` but SQLCipher is missing.
- Added tests for encrypted-mode failure and explicit plaintext fallback.
