# P5-T5.10 - Standardize cache warming status ownership

## Audit reference

- N3: `cache_warming_status` treated a background worker as the object that owned warming status, even though `get_warming_status()` belongs to `CacheWarmer`.

## Changes completed

- Introduced a `WarmingStatusProvider` protocol and explicit `set_cache_warmer()` setter for the cache status tool.
- Kept `set_background_worker()` as a compatibility setter for providers that proxy warming status.
- Added canonical inactive warming status payloads with the same field names used by `CacheWarmer`.
- Added `BackgroundWorker.get_warming_status()` and `set_cache_warmer()` so a worker can proxy an attached warmer or report inactive warming status.
- Updated tool docs and tests to use `operations_total`, `operations_completed`, `operations_failed`, and `progress_percent` consistently.
