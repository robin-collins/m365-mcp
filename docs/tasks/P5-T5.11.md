# P5-T5.11 - Wire cache warming and background refresh only after hardening

## Audit reference

- N2: cache warming and background refresh code existed but was not connected to server lifecycle, status reporting, or clean shutdown.

## Changes completed

- Added a cache runtime lifecycle in `server.py` that starts `BackgroundWorker` and `CacheWarmer` only when `CACHE_WARMING_ENABLED` is true.
- Wrapped stdio, HTTP, OAuth, and bearer-token transports so enabled cache services start with the server and stop cleanly on shutdown.
- Wired `cache_warming_status` to the worker provider when enabled, with the worker proxying its attached warmer status.
- Added `CacheWarmer.stop()` so in-flight warming tasks can be cancelled during shutdown.
- Added stale-cache refresh enqueueing when warming/background refresh is enabled, with duplicate queued/running refresh suppression.
- Kept the default disabled flag path unchanged: no cache manager or worker is created.
