# P5-T5.6 - Recover cleanly from cache key mismatch or corruption

## Audit reference

- Finding 4: encrypted cache startup could crash permanently when a stored cache used a stale key or the database file was corrupt.

## Changes completed

- Added recoverable database-error detection for corrupt, malformed, encrypted-with-wrong-key, and HMAC failure messages.
- Updated cache initialization to retry schema creation once after closing pooled handles and deleting the bad cache database plus WAL/SHM sidecars.
- Closed partially initialized SQLite handles when PRAGMA setup fails, which is required for Windows-safe deletion.
- Added regressions for both a corrupt cache file and an encrypted cache opened with a different key.
