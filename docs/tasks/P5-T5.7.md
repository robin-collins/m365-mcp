# P5-T5.7 - Warn when an ephemeral cache key is used

## Audit reference

- Finding 4: generated cache keys that cannot be persisted can make an encrypted cache unreadable on restart without any clear operator warning.

## Changes completed

- Updated `EncryptionKeyManager.get_or_create_key()` to log a clear warning when no keyring key, no `M365_MCP_CACHE_KEY`, and no successful keyring write provide durable storage.
- Kept key material out of logs while explaining that encrypted cache files may be unreadable after restart.
- Added log assertions for storage failure and missing-keyring paths, plus negative assertions for keyring and environment-provided durable keys.

## Verification

- `uv run pytest tests/test_encryption.py -q`
  - Result: 33 passed.
- `uvx ruff check src/m365_mcp/encryption.py tests/test_encryption.py`
  - Result: passed.

## Notes

- The warning is emitted only when the returned generated key is ephemeral; durable key sources remain quiet.
