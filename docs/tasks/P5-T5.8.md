# P5-T5.8 - Parameterize or safely encode SQLCipher `PRAGMA key`

## Audit reference

- N5: SQLCipher keys were interpolated directly into `PRAGMA key` statements.

## Changes completed

- Verified this SQLCipher build rejects bound parameters for `PRAGMA key`, so the implementation now uses a shared raw hex-key encoder instead.
- Added `EncryptionKeyManager.sqlcipher_key_pragma()` to validate a base64 256-bit key and emit SQLCipher raw-key syntax without passphrase interpolation.
- Updated cache connection setup and cache migration detection/migration to use the shared encoded PRAGMA.
- Added tests for generated keys, environment-style base64 keys, invalid key rejection, and migration PRAGMA use.
