# P5-T5.9 - Make background task claiming atomic

## Audit reference

- Finding 6: background workers selected a queued task and marked it running in separate operations, allowing duplicate claims under concurrency.

## Changes completed

- Replaced the worker's separate select-then-update flow with a single `UPDATE ... RETURNING` claim.
- Removed the redundant follow-up status update to `running`; claiming now sets `started_at` and returns the claimed task atomically.
- Added a two-worker concurrency regression proving a single queued task is executed once.
