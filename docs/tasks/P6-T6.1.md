# P6-T6.1 - Escape OData contact-search string literals

## Audit reference

- N12: contact search interpolated raw query text into an OData `$filter` string.

## Changes completed

- Added `_odata_string_literal()` to encode OData single-quoted string literals by doubling apostrophes.
- Updated `_search_contacts_filter()` to use the escaped literal for `displayName`, `givenName`, and `surname` prefix filters.
- Added regressions for apostrophes, already-doubled quotes, and an injection-shaped filter fragment.
- Made two existing search tests self-contained by stubbing account-type lookup when run in isolation.
