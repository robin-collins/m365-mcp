# P6-T6.2 - Validate `emailrules_create` like `emailrules_update`

## Audit reference

- N13: `emailrules_create` sent caller-provided rule payloads to Graph without the validation already present on `emailrules_update`.

## Changes completed

- Added shared display-name and sequence validators.
- Updated `emailrules_create` to trim/validate display name, sequence, enabled flag, conditions, actions, and exceptions before calling Graph.
- Reused the existing predicate/action normalization path so create and update now produce the same normalized payload shapes.
- Added tests for valid create normalization and invalid create payload rejection before any Graph call.
