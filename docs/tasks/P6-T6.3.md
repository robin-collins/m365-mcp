# P6-T6.3 - Prevent blocking device-flow auth inside normal MCP requests

## Audit reference

- Finding 9: `get_token()` could fall into blocking device-code authentication during ordinary MCP tool requests.

## Changes completed

- Added `M365_MCP_INTERACTIVE_AUTH` as the explicit opt-in for device-code flow inside `get_token()`.
- Made `get_token()` fail fast with an actionable message when silent token acquisition misses in normal server context.
- Set `M365_MCP_INTERACTIVE_AUTH=true` in `authenticate.py` before importing the auth module.
- Added tests for server-context fail-fast behavior, explicit interactive allowance, and the standalone authentication script opt-in.

## Verification

- `uv run pytest tests/test_account_validation.py -q`
  - Result: 9 passed.
- `uvx ruff check src/m365_mcp/auth.py authenticate.py tests/test_account_validation.py`
  - Result: passed.

## Notes

- Non-blocking MCP account-auth tools still initiate and complete device flow explicitly; the fail-fast guard applies to token acquisition during normal Graph calls.
