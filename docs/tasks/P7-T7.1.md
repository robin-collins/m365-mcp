# P7-T7.1 - Decide whether to delete or wire cache migration utilities

## Audit reference

- N6: cache migration utilities were present but not wired into real cache startup.

## Decision

- Deleted the migration utility instead of wiring it. Phase 5 now handles incompatible or corrupt cache databases by closing handles, deleting the bad cache files, and recreating an empty cache. The cache is disposable data, so keeping a separate unwired migration path added risk without runtime value.

## Changes completed

- Removed `src/m365_mcp/cache_migration.py`.
- Removed the migration-only test module.
- Removed `cache_migration.py` from structure/file-tree documentation.
- Confirmed no remaining repository references to `cache_migration`, `detect_and_migrate`, or `migrate_to_encrypted_cache`.
