# P7-T7.2 - Fix `cache_invalidate` account filtering

## Audit reference

- N7: `cache_invalidate` rewrote wildcard patterns to include `account_id` but then failed to pass the actual account filter to `CacheManager.invalidate_pattern()`.

## Changes completed

- Removed fragile pattern rewriting from `cache_invalidate`.
- Passed `account_id` directly to `invalidate_pattern()`.
- Added tool-level regressions for account-scoped wildcard invalidation and account-scoped exact-pattern invalidation.

## Verification

- `uv run pytest tests/test_cache_tools.py -q -k "invalidate"`
  - Result: 5 passed, 14 deselected.
- `uvx ruff check src/m365_mcp/tools/cache_tools.py tests/test_cache_tools.py`
  - Result: passed.

## Notes

- Returned `pattern` now remains the caller-provided pattern, while account scoping is enforced by the cache manager query.
