# P7-T7.3 - Resolve the account-id-first signature drift

## Audit reference

- N15: 47 account-scoped MCP tools expose `account_id` somewhere other than the first parameter, conflicting with the prior steering rule.

## Decision

- Option A: preserve current public signatures and update steering docs to describe the real API. Reordering 47 established MCP tool signatures would be a breaking change for generated clients, saved tool calls, and existing users.

## Changes completed

- Updated `.projects/steering/mcp-server.md` to require `account_id` on account-scoped tools while preserving existing public parameter order.
- Added a changelog note documenting the compatibility policy.
- Added `tests/test_tool_signature_policy.py`, an AST policy test that locks the current non-first `account_id` set and reports zero unexpected drift.
