# P8-T8.5 Release / PR Notes

## Status

Completed.

## User-Visible Behavior Changes

- Normal MCP requests now fail fast when no cached Microsoft token is available
  instead of starting a blocking device-code flow.
- `uv run authenticate.py` remains the explicit interactive authentication path.
- Cache warming and stale-cache refresh are fully wired but remain opt-in behind
  `M365_MCP_CACHE_WARMING=true`.
- `cache_invalidate(pattern, account_id=...)` now scopes invalidation by the
  requested account for wildcard and exact patterns.
- Public tool signatures preserve existing parameter order for compatibility;
  account-scoped tools still expose `account_id`.

## Security And Reliability Fixes

- SQLCipher is required when cache encryption is enabled; encrypted cache
  startup no longer silently falls back to plaintext SQLite.
- SQLCipher connection settings and raw-key PRAGMA generation now use the
  configured hardened path.
- Cache connection pooling is thread-safe, failed connections are discarded, and
  `CacheManager.close()` explicitly releases handles.
- Cache initialization recovers from corrupt databases and encryption key
  mismatches by recreating the cache.
- Generated cache encryption keys now warn when they cannot be persisted.
- Background cache task claiming is atomic across workers.
- Contact search OData literals are escaped.
- `emailrules_create` validates payloads consistently with update behavior.
- Dead cache migration utilities and the shadowed root tool registry were
  removed.

## Documentation And Compatibility

- Added root `QUICKSTART.md` and `SECURITY.md`.
- Refreshed README, cache docs, FILETREE, `.env.example`, and steering docs for
  current tool names, cache behavior, authentication behavior, and signature
  policy.
- Added regression coverage for the current public tool surface and signature
  compatibility policy.
